## Sub-Phase Completion Gate (Human Verification & Satisfaction)
Sub-phase completion is ALWAYS review-driven, even if the project's
default autonomy mode is agent-driven or sweet-spot. Before marking
any sub-phase complete in task.md:
1. Produce a Verification Summary artifact (metrics, regression
   results, open postmortems, recording/screenshots, self-assessment).
2. Stop and wait for one of: PROCEED, REVISE, ROLLBACK, HOLD,
   ESCALATE, OVERRIDE-COMPLETE.
3. Never infer approval from silence or unrelated conversation.
4. Do not begin the next sub-phase's Step 1 until PROCEED is received
   for the current one.
